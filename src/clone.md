# 克隆远程

````admonish reset title="重置您的进度" collapsible=true
要将您的进度重置到本章开头，请运行以下命令：

```sh
curl https://jj-for-everyone.github.io/reset.sh | bash -s clone
cd ~/jj-tutorial/repo
```
````

在本教程的开头，我们通过使用 `jj git init` 初始化一个全新的项目开始了我们的旅程。
然后我们后来添加了一个远程作为备份解决方案。
实际上，情况通常是反过来的：
您需要处理一个已经存在的仓库。

让我们通过删除主仓库来模拟这种情况：

```sh
cd ~
rm -rf ~/jj-tutorial/repo
```

现在，我们只剩下远程仓库了。
我们如何才能获得它的"本地"副本以便继续工作？
获取仓库副本的行话是**克隆**它。
相关的 Jujutsu 命令也相应地命名：

```sh
jj git clone ~/jj-tutorial/remote ~/jj-tutorial/repo
```

最后两个参数是 (1) 克隆的**源**和 (2) **目标**——存储克隆仓库的位置。
当您从远程克隆时，会自动使用默认名称 `origin` 连接该远程。

对于教程，我们还需要重新创建本仓库特定的作者配置：

```sh
cd ~/jj-tutorial/repo
jj config set --repo user.name "Alice"
jj config set --repo user.email "alice@local"
jj metaedit --update-author
```
