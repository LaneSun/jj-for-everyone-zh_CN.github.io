哦不！
看起来 Bob 的研究结果被记录到了提交中。
压缩包 `submission_alice_bob.tar.gz` 不应该被版本控制跟踪。

默认情况下，Jujutsu 会将项目中的每一个文件都记录到你的工作副本中，这通常正是你想要的。
但有时，你需要告诉 Jujutsu 忽略某些文件并将其排除在记录之外。
为此，如果项目中存在 `.gitignore` 文件，Jujutsu 会读取它。
该文件可以包含文件名、目录和模式的列表，用于描述不应被版本控制跟踪的文件集合。

为了解决这个问题，Bob 在 `.gitignore` 文件中添加了一个压缩包的模式：

```sh
echo "*.tar.gz" > .gitignore
```

星号 `*` 符号是一个特殊的 [glob](https://en.wikipedia.org/wiki/Glob_(programming)) 字符，它可以展开匹配任何文件名，包括 `submission_alice_bob`。
只将精确的文件名 `submission_alice_bob.tar.gz` 添加到 `.gitignore` 文件也可以，但上面的模式可以处理_任何_压缩包。

````admonish info title="全局忽略文件" collapsible=true
有时你可能在多个项目中都有相同的仅供个人使用的文件。
这种情况下，全局忽略这些文件而不仅仅是针对特定项目是有意义的。
将个人文件添加到你所做的每一个项目的 `.gitignore` 中会有点繁琐。

对 macOS 用户来说有一个相关的例子：
Finder 应用程序会在用它打开的目录中存储名为 `.DS_Store` 的隐藏文件。
这些文件可能出现在任何地方，基本上永远不应该被版本控制。
如果你是 macOS 用户，考虑将 `.DS_Store` 添加到你的全局 `.gitignore` 文件中：

```sh
echo ".DS_Store" >> ~/.gitignore
```
````

嗯... `.gitignore` 文件加入了，但压缩包_仍然_被记录在提交中。
原因在于 Jujutsu 只会对尚未开始跟踪的文件考虑 `.gitignore`。
由于 Bob 在不想要的文件已开始被跟踪之后才添加 `.gitignore`，所以它没有生效。
幸运的是，Jujutsu 有一个命令可以使其停止跟踪已被跟踪的文件：

```sh
jj file untrack submission_alice_bob.tar.gz
```

`untrack` 命令只有在要取消跟踪的文件已存在于 `.gitignore` 中时才有效。
否则，Jujutsu 会在取消跟踪后立即重新开始跟踪该文件。
请注意，你也可以一次性忽略和取消跟踪整个目录。
